XML

HISTORIA

El lenguaje XML tiene sus raíces en el GML (General Markup Language) creado por IBM en los años 70. Este surgió para abordar la necesidad de almacenar grandes cantidades de información de diversas áreas en la empresa. GML permitía clasificar y escribir documentos de manera que pudieran ser procesados adecuadamente. La ISO adoptó este lenguaje y creó el SGML (Standard Generalized Markup Language) en 1986. SGML, siendo una versión estándar de GML, era versátil y capaz de adaptarse a diversos problemas, sirviendo como base para el desarrollo de sistemas de almacenamiento de información. En 1989, en el contexto de Internet, surgió el HTML (Hypertext Markup Language) como un nuevo lenguaje creado por un usuario que había trabajado con lenguajes de etiquetas e hiperenlaces. HTML se utilizó para la World Wide Web y fue rápidamente adoptado, aunque su crecimiento descontrolado generó desafíos. En 1998, el W3C (World Wide Web Consortium) comenzó el desarrollo del XML (Extended Markup Language) con el objetivo de abordar las limitaciones del HTML en el tratamiento de la información. XML se diseñó de manera más cuidadosa y ha continuado evolucionando con la colaboración de expertos en la materia. Intenta resolver problemas del HTML, como la mezcla de contenido y estilos, la falta de compatibilidad con diversos dispositivos y la dependencia de la presentación en pantalla del visor utilizado. Aunque el W3C ha buscado establecer reglas y estándares para el HTML, el XML se ha perfilado como una alternativa más estructurada y versátil para la representación y el intercambio de datos.

NORMAS DE USO Y SINTAXIS

Estas son las reglas que se deben seguir al crear la sintaxis XML:

  • Todos los elementos XML deben tener una etiqueta de cierre.
  • Las etiquetas XML son sensibles a mayúsculas y minúsculas.
  • Todos los elementos XML deben estar debidamente anidados.
  • Todos los documentos XML deben tener un elemento raíz.
  • Los valores de atributo siempre deben estar entre comillas.

No está permitido omitir la etiqueta de cierre al crear la sintaxis XML. Los elementos XML deben tener una etiqueta de cierre. Cuando crea documentos XML, la etiqueta <Body> es diferente de la etiqueta <body>. Un anidamiento incorrecto de las etiquetas no tiene sentido para XML. Todos los documentos XML deben contener un par de etiquetas individuales para definir un elemento raíz. Todos los demás elementos deben estar dentro de este elemento raíz. Todos los elementos pueden tener subelementos (elementos hijo). Los subelementos deben estar anidados correctamente dentro de su elemento padre. No está permitido omitir las comillas alrededor de los valores de atributos. Los elementos XML pueden tener atributos en pares nombre/valor; sin embargo, el valor de atributo siempre debe estar entre comillas.

APLICACIONES EN EL ÁMBITO EMPRESARIAL

Intercambio de Datos:

  • JSON es ampliamente utilizado para el intercambio de datos entre clientes y servidores en aplicaciones web y servicios web.

APIs (Interfaz de Programación de Aplicaciones):

  • Muchas APIs utilizan JSON para la transmisión de datos debido a su legibilidad y simplicidad.

Configuración y Almacenamiento de Datos:

  • Se utiliza para almacenar configuraciones y datos estructurados en sistemas empresariales.

Formato de Registro:

  • JSON se usa en registros de eventos y sistemas de registro para almacenar información de forma estructurada.

Integración de Sistemas:

  • Facilita la integración de sistemas heterogéneos, ya que es fácil de entender y procesar en diferentes plataformas.

Bases de Datos NoSQL:

  • Algunas bases de datos NoSQL utilizan JSON como formato para almacenar y consultar datos.
